# ✦ Spark — MVP 기획안

> 버전 3.4 | 2026년  
> Co-writing 글쓰기 앱 | "생각에 불을 붙이는 글쓰기"

---

## 컨텍스트 인계

너는 전문적인 PM이자 개발자야. MVP를 만들기 위해 최대한 단순하고 간소화된 방향으로 개발해야 해. 내 의견을 항상 비판적으로 분석해야 해.

## 1. 기획 진화 흐름

| 단계 | 핵심 변화 | 문제 / 이유 |
|---|---|---|

## 2. 제품 개요

### 핵심 가치
- **시작을 쉽게** — 글감 + 첫 문장이 질문 형식으로 제시됨
- **이탈하지 않게** — 편리한 UX로 흐름 유지
- **쌓이게** — 글이 누적되어 습관화 + 시각적으로 보임

### 궁극적 목적
정해진 글감에서 출발하지만, 궁극적으로 사용자가 자신의 생각을 구조화하고 이를 글로 표현해내게 만드는 것. 편의 서비스를 제공하되, 자유 주제 글쓰기도 지원.

### 타겟 사용자
| 구분 | 내용 |
|---|---|
| 연령대 | 20대 중반 ~ 30대 중반 |
| 타입 1 | 스타트업 종사자 / 커리어 성장에 관심 있는 직장인 |
| 타입 2 | 창업을 희망하는 사람 (막연히 꿈꾸는 직장인 + 예비 창업자) |
| 타입 3 | 나의 내면을 잘 가꾸어내고 싶은 사람 |
| 페인포인트 | 내 생각을 구조화하는 데 어려움이 있어 글쓰기로 키우고 싶은 사람 |

### 비전 로드맵
| 단계 | 내용 |
|---|---|
| MVP (지금) | 글감 + 첫 문장으로 쓰기 시작 → 습관 형성 |
| v2 | 3초 멈춤 → 다음 질문 제안 → 생각 전개 훈련 |
| v3 | 글 구조 피드백 (주장·근거·구조화) → 기획력 훈련 |

### 5-1. 서비스 톤앤매너

**한 줄 정의:** 카페에서 나를 잘 아는 친구가 건네는 말

| 항목 | 기준 |
|---|---|
| 경어 | 해요체 — 합쇼체 X / 반말 X |
| 온도 | 차분하고 따뜻함 — 열정적·명령적 금지 |
| 거리감 | 옆에 앉은 친구 — 코치·선생님·AI 느낌 X |
| 문장 길이 | 짧고 끊어지는 리듬 — 길게 이어지지 않음 |
| 느낌표 | 최소 1개 이하 — 과용 금지 |
| 금지 표현 | "~해보세요!" / "~하면 좋아요!" / "~하실 수 있어요" / AI 챗봇 말투 |
| 버튼 레이블 원칙 | 기능 설명 X → 유저의 심리·행동으로 표현 |
| 넛지 원칙 | 압박 X → 허락·공감으로 ("찾고 계신가요?" X → "그냥 써도 괜찮아요" O) |
| 핵심 테스트 | "이 문장을 카페에서 친한 후배한테 말할 수 있나?" |

**레퍼런스:** Headspace (차분함·따뜻함) + Duolingo (친근함) — 단, Duolingo의 위트·과열은 제외

### 5-2. 글감 톤앤매너

**핵심 목적:** 읽는 순간 "나도 이런 적 있는데" 또는 "나는 어떻지?"가 즉각 튀어나오는 글쓰기 출발점

| 항목 | 기준 |
|---|---|
| 형태 | 2인칭 질문형 고정 — 독백형 서술 X |
| 어미 | "~한 적 있나요?" / "~이었나요?" / "~했나요?" |
| 주어 | 생략 권장 — "당신은" 사용 시 어색함 주의 |
| 길이 | 15~25자 이내 |
| 구성 비율 | 감정·탐구형 70% + 구조화 유도형 30% |
| 금지 | 번아웃·탈진·무기력·결핍·후회 전제 / 뻔한 자기계발 문구 / 특정 직군·나이·상황 한정 / 결론을 내버리는 글감 |

**좋은 글감 예시**
```
"잘하고 싶은 마음이 오히려 발목을 잡은 적 있나요?"
"성장을 원했나요, 인정을 원했나요?"
"처음 해봤는데 생각보다 잘 됐던 일이 있나요?"
```

**나쁜 글감 예시**
```
"번아웃이 왔을 때 어떻게 극복했나요?" → 부정적 상태 전제
"내가 롤모델을 잘못 고른 이유는?" → 독백형 + 결론 전제
"직장인이라면 한 번쯤 느끼는 그 감정" → 특정 집단 한정
```

### 5-3. 힌트 기준점

**결정 (v3.3):** 힌트 제거

글감이 2인칭 질문형으로 바뀌면서 글감 자체가 이미 충분한 출발점이 됨.
힌트는 오히려 질문의 여운을 끊고 유저의 글을 특정 방향으로 좁힐 수 있어 제거.

**대체:** 에디터 입력창 placeholder
```
"떠오르는 장면부터 써봐요."
```
심리적 허락만 주는 한 줄. 장황한 힌트 없이 빈 화면의 부담을 덜어줌.

**제거된 것들**
- `POST /api/first-sentences` 스트리밍 API
- `topics.hint` 컬럼 추가 계획
- 힌트 1,600개 생성 및 검수 시스템

### 5-4. 카테고리 경계 기준

**핵심 원칙:** 카테고리는 글감의 탐구 방향을 정하는 것이지, 소재를 제한하는 것이 아님

| 카테고리 | 한 줄 정의 | 핵심 구분 |
|---|---|---|
| 루틴 | 반복되는 행동 패턴 그 자체 | 개별 습관·행동에 집중 |
| 라이프스타일 | 내가 사는 방식의 전체적인 결 | 삶의 방식·가치관 전체 |
| 웰니스 | 몸과 마음의 상태·신호·회복 | 몸·마음의 감각과 회복 |
| 감정 | 내가 느끼는 것을 탐구 | 감정을 데이터로 관찰 |
| 경험 | 겪은 것에서 발견 | 과거 경험을 탐구로 |
| 커리어 | 일하는 나를 탐구 | 직업적 자아 관찰 |

> 나머지 카테고리는 운영하면서 보완

### 5-5. 글감 품질 기준 (검수용)

**통과 조건 — 아래 3개 모두 충족해야 함**
```
1. 읽는 순간 자기 경험이 떠오르는가?
2. 특정 직군·나이·상황을 전제하지 않는가?
3. 결론이나 교훈을 미리 내버리지 않는가?
```

**탈락 조건 — 하나라도 해당하면 탈락**
```
1. 부정적 상태(번아웃·무기력·결핍·후회)를 전제함
2. 뻔한 자기계발 문구 ("더 나은 나를 위해" 등)
3. 해석이나 설명이 이어져야 완결되는 글감
4. 같은 카테고리 내 다른 글감과 내용·느낌이 겹침
```

## 6. Claude API 프롬프트 방향

### 공통 프롬프트 원칙 (16개 공유)
- 독자: 20대 후반~30대 / 커리어 성장·창업 관심·내면 탐구
- 글감 정의: 읽는 순간 "나도 이런 적 있는데" 또는 "나는 어떻지?"가 즉각 튀어나오는 출발점
- 톤: 긍정·호기심·발견 / 구체적 장면
- 금지: 부정·결핍·번아웃·탈진·무기력 전제 / 뻔한 자기계발 / 추상적 감정 / 결론 내버리기
- 구성 비율: 감정·탐구형 70% + 구조화 유도형 30%

### 글감 스크립트 버전 이력 (`scripts/generate-topics.mjs`)

| 스크립트 버전 | 변경 내용 |
|---|---|
| s1.0 | 단일 프롬프트 / 카테고리 10개 / 첫 문장 DB 저장 방식 |
| s2.0 | 카테고리 16개 / 카테고리별 개별 프롬프트 (tone·examples·forbidden) / COMMON_RULES 분리 |
| s2.1 | 루틴·라이프스타일·웰니스 카테고리 경계 명확화 (forbidden에 서로 교차 금지 추가) |
| s2.2 | 목표 개수 자동 계산 (`100 - 기존 개수`만 생성) / 질문형 비율 강화 (글감 40% 이상, 첫 문장 3개 중 2개) |
| s2.3 | 첫 문장 생성 제거 → 온디맨드 스트리밍 전환 (`POST /api/first-sentences`) |
| s2.4 | 배치 API 도입 (비용 50% 절감) / 기존 글감 최근 20개만 주입 (토큰 절감) / custom_id 영문 매핑 추가 |
| s3.0 | 글감 톤 전면 변경 — 독백형 서술 → 2인칭 질문형 / 힌트 제거 결정으로 `first_sentences` 관련 로직 완전 삭제 / 에디터 placeholder로 대체 |

### 첫 문장 흐름 변경 (v3.3)

| 구분 | 기존 | 변경 |
|---|---|---|
| 힌트 방식 | 스트리밍 API로 실시간 생성 | **제거** |
| 대체 | 없음 | 에디터 입력창 placeholder: "떠오르는 장면부터 써봐요." |
| 이유 | 글감 2인칭 질문형 전환으로 글감 자체가 충분한 출발점이 됨 | |

### 첫 문장 적용 흐름

| 구분 | 기존 | 변경 (s2.3~) |
|---|---|---|
| 생성 시점 | 글감 생성 시 미리 만들어서 DB 저장 | 유저가 글감 선택하는 순간 실시간 생성 |
| 방식 | 일괄 저장 | 스트리밍 (타이핑되듯 한 글자씩 나타남) |
| 비용 | ~$3.70 (1,600회 API 호출) | ~$0.04 (실제 사용할 때만 과금) |
| API | 없음 | `POST /api/first-sentences` |

```
유저가 글감 선택
       ↓
POST /api/first-sentences 호출
       ↓
Claude가 첫 문장 3개를 스트리밍으로 생성
       ↓
화면에 타이핑되듯 표시
       ↓
유저가 마음에 드는 첫 문장 선택 → 에디터로 진입
```

## 6. DB 스키마

| 테이블 | 주요 컬럼 | RLS |
|---|---|---|
| users | id, email, onboarding_completed | 본인만 |
| topics | id, content | 전체 공개 |
| writings | id, user_id, topic_id, topic_content, first_sentence, body | 본인만 |
| user_seen_topics | id, user_id, topic_id | 본인만 |
| topic_categories | topic_id, category | 전체 공개 |
| user_categories | user_id, category, click_order, is_removed, selected_at | 본인만 |

### writings 테이블 비고
- `topic_id`가 null이면 자유 주제 글 (category = '자유')
- `topic_content`는 유저가 수정한 글감 텍스트 (원본 ID는 `topic_id`로 고정)

### topics 테이블 비고
- `first_sentences` 컬럼 제거 (v3.1) — 첫 문장은 DB에 미리 저장하지 않음
- 유저가 글감을 선택하는 순간 `POST /api/first-sentences`로 실시간 스트리밍 생성

## 9. 홈 화면 UX (🔧 재작성 예정)

### 벤치마킹
- **세계문학 일력 (민음사)**: 매일 채워지는 칸 → "빈 칸을 채우고 싶은" 심리 유도
- **Duolingo 스트릭**: 7일 스트릭 달성 유저는 미달성 유저보다 다음날 재방문 확률 2.4배. Day 0~7 구간이 습관 형성의 핵심 구간
- **Endowed Progress Effect**: 목표의 일부가 미리 채워져 있을 때 완주 욕구가 강해짐
- **Hinge 카드 UX**: 선택지 1개는 이탈률 높음 / 3개 이상은 선택 피로 → 스와이프 방식으로 절충

### 확정 UX

| 항목 | 확정 내용 |
|---|---|
| 네비게이션 | 탭바 제거 → **햄버거 사이드바** (글쓰기 몰입 방해 제거) |
| 캘린더 | 기본: 오늘이 포함된 1주만 표시 / 토글 → 전체 월 펼침 |
| 캘린더 표시 | 쓴 날 = 채워진 검은 원 / 오늘 = 테두리 원 / 빈 날 = 연한 원 |
| 글감 카드 구조 | 옆 카드가 살짝 보이는 **가로 슬라이드** |
| 글감 개수 | 개수 제한 없는 무한 슬라이드 (본 글감 제외) |
| 글감 뽑기 방식 | 선택 카테고리 기반 Fisher-Yates 셔플 |
| 넛지 | 5~6개 넘기면 등장: "그냥 써도 괜찮아요" + **"편하게 써볼게요"** 버튼 |
| 무작위 버튼 동작 | 바로 에디터 진입 X → 랜덤 글감 카드 1개 표시 후 "이걸로 쓸게요" 확인 |
| 자유 주제 | 글감 카드 아래 **항상** 노출 — 점선 텍스트 링크 "그냥 쓰고 싶어요" |
| 카테고리 편집 | 사이드바 내 "카테고리 편집" 항목 → 온보딩 화면 재활용 |
| 글 쓴 직후 피드백 | 홈 복귀 시 캘린더 오늘 칸 채워짐 + "오늘 칸이 채워졌어요" 표시 |
| 최대 너비 | 390px 중앙 고정 |

### 글감 카드 슬라이드 흐름
```
[글감 카드] [글감 카드] [글감 카드 살짝 보임]
                ↓ 5~6개 넘기면
"그냥 써도 괜찮아요"
[ 편하게 써볼게요 ]   ← 무작위 버튼 (랜덤 카드 1개 확인 후 진입)

─────────────────────────────
그냥 쓰고 싶어요             ← 자유 주제 (항상 표시, 점선 텍스트 링크)
```

### 사이드바 구성
```
spark
─────────────
✦ 글감 홈         (현재 화면)
☰ 내 글 (책장)
⊙ 설정
─────────────
카테고리 편집
로그아웃
```

```

                                                        └─ 내 글 → [아카이브 책장]
                                                                       └─ 카드 클릭 → [에디터 (수정)]
```

### 유저 흐름 점검 결과 (비판적 검토)

#### 🔴 해결 필요
| 문제 | 해결 방향 |
|---|---|
| 글 쓴 직후 피드백 없음 | 홈 복귀 시 캘린더 오늘 칸 채워지는 즉각 피드백 |

| 글감 소진 시 UX 없음 | "모든 글감을 봤어요. 다시 보기" 옵션 추가 필요 |

#### 🟡 보완 필요
| 문제 | 해결 방향 |
|---|---|
| Day 1 경험 빈약 (책장·캘린더 모두 비어있음) | 첫 진입 시 간단한 온보딩 힌트 or 빈 상태 안내 문구 |

- middleware 접근 제어 완벽
