# ✦ Spark — MVP 기획안

> 버전 3.0 | 2026년  
> Co-writing 글쓰기 앱 | "생각에 불을 붙이는 글쓰기"

---

## 컨텍스트 인계

너는 전문적인 PM이자 개발자야. MVP를 만들기 위해 최대한 단순하고 간소화된 방향으로 개발해야 해. 내 의견을 항상 비판적으로 분석해야 해.

## 1. 기획 진화 흐름

| 단계 | 핵심 변화 | 문제 / 이유 |
|---|---|---|
| 1단계 초기 설계 | 카테고리 10개 / 결핍·후회 중심 톤 | 부정적 감정 편향 / 카테고리 중복 |
| 2단계 타겟 재정의 | 직장인 한정 → 예비 창업자·내면 탐구로 확장 | 카테고리와 글감의 결 전면 재검토 필요 |
| 3단계 글감 설계 원칙 | 긍정·호기심·탐구 톤 확립 | 목적 추가: 글쓰기 습관 + 생각 구조화 훈련 |
| 4단계 카테고리 재편 | 10개 → 16개 (4개 축) | 중복 통합·분리·확장 |
| 5단계 프롬프트 방향 | 단일 → 카테고리별 16개 개별 설계 | 각 카테고리마다 톤·예시·금지어 따로 설계 |
| 6단계 UX 재설계 | 저널/일력 벤치마킹 → 책장 아카이브 도입 | 습관 시각화 + 수집욕 기반 리텐션 설계 |

## 2. 제품 개요

### 핵심 가치
- **시작을 쉽게** — 글감 + 첫 문장이 질문 형식으로 제시됨
- **이탈하지 않게** — 편리한 UX로 흐름 유지
- **쌓이게** — 글이 누적되어 습관화 + 시각적으로 보임

### 궁극적 목적
정해진 글감에서 출발하지만, 궁극적으로 사용자가 자신의 생각을 구조화하고 이를 글로 표현해내게 만드는 것. 편의 서비스를 제공하되, 자유 주제 글쓰기도 지원.

### 타겟 사용자
| 구분 | 내용 |
|---|---|
| 연령대 | 20대 중반 ~ 30대 중반 |
| 타입 1 | 스타트업 종사자 / 커리어 성장에 관심 있는 직장인 |
| 타입 2 | 창업을 희망하는 사람 (막연히 꿈꾸는 직장인 + 예비 창업자) |
| 타입 3 | 나의 내면을 잘 가꾸어내고 싶은 사람 |
| 페인포인트 | 내 생각을 구조화하는 데 어려움이 있어 글쓰기로 키우고 싶은 사람 |

### 비전 로드맵
| 단계 | 내용 |
|---|---|
| MVP (지금) | 글감 + 첫 문장으로 쓰기 시작 → 습관 형성 |
| v2 | 3초 멈춤 → 다음 질문 제안 → 생각 전개 훈련 |
| v3 | 글 구조 피드백 (주장·근거·구조화) → 기획력 훈련 |

## 4. 글감 설계 원칙

- 톤: 긍정·호기심·탐구 / 번아웃·탈진·무기력 전제 금지
- 구성: 감정·탐구형 70% + 구조화 유도형 30%
- 형태: 서술형 + 질문형 병행 / 구체적 장면 포착

---

## 5. Claude API 프롬프트 방향

### 공통 프롬프트 원칙 (16개 공유)
- 독자: 20대 후반~30대 / 커리어 성장·창업 관심·내면 탐구
- 글감 정의: 읽는 순간 "나도 이런 적 있는데" 또는 "나는 어떻지?"가 즉각 튀어나오는 출발점
- 톤: 긍정·호기심·발견 / 구체적 장면
- 금지: 부정·결핍·번아웃·탈진·무기력 전제 / 뻔한 자기계발 / 추상적 감정 / 결론 내버리기
- 구성 비율: 감정·탐구형 70% + 구조화 유도형 30%

## 6. DB 스키마

| 테이블 | 주요 컬럼 | RLS |
|---|---|---|
| users | id, email, onboarding_completed | 본인만 |
| topics | id, content, first_sentences(array) | 전체 공개 |
| writings | id, user_id, topic_id, topic_content, first_sentence, body | 본인만 |
| user_seen_topics | id, user_id, topic_id | 본인만 |
| topic_categories | topic_id, category | 전체 공개 |
| user_categories | user_id, category, click_order, is_removed, selected_at | 본인만 |

### writings 테이블 비고
- `topic_id`가 null이면 자유 주제 글 (category = '자유')
- `topic_content`는 유저가 수정한 글감 텍스트 (원본 ID는 `topic_id`로 고정)

## 9. 홈 화면 UX (🔧 재작성 예정)

### 벤치마킹
- **세계문학 일력 (민음사)**: 매일 채워지는 칸 → "빈 칸을 채우고 싶은" 심리 유도
- **Duolingo 스트릭**: 7일 스트릭 달성 유저는 미달성 유저보다 다음날 재방문 확률 2.4배. Day 0~7 구간이 습관 형성의 핵심 구간
- **Endowed Progress Effect**: 목표의 일부가 미리 채워져 있을 때 완주 욕구가 강해짐

### 확정 UX

| 항목 | 확정 내용 |
|---|---|
| 네비게이션 | 탭바 제거 → **햄버거 사이드바** (글쓰기 몰입 방해 제거) |
| 캘린더 | 기본: 오늘이 포함된 1주만 표시 / 토글 → 전체 월 펼침 |
| 캘린더 표시 | 쓴 날 = 채워진 검은 원 / 오늘 = 테두리 원 / 빈 날 = 연한 원 |
| 글감 추천 | 선택 카테고리 기반 랜덤 3개 (본 글감 제외) |
| 글감 뽑기 방식 | 카테고리 Fisher-Yates 셔플 후 카테고리당 1개씩 |
| 새로고침 | "다른 글감 보기" 텍스트 버튼 |
| 자유 주제 | 글감 카드 아래 점선 버튼 "✎ 자유 주제로 쓰기" |
| 카테고리 편집 | 사이드바 내 "카테고리 편집" 항목 → 온보딩 화면 재활용 |
| 글 쓴 직후 피드백 | 홈 복귀 시 캘린더 오늘 칸 채워짐 + "오늘 칸이 채워졌어요" 표시 |
| 최대 너비 | 390px 중앙 고정 |

### 사이드바 구성
```
spark
─────────────
✦ 글감 홈         (현재 화면)
☰ 내 글 (책장)
⊙ 설정
─────────────
카테고리 편집
로그아웃
```

### 미해결 이슈
- **글감 소진 시 UX 미정**: 카테고리당 100개 글감이 소진되면 "모든 글감을 봤어요. 다시 보기" 옵션 필요

## 11. 아카이브 UX (🔧 다음 단계)

### 벤치마킹
- **세계문학전집 앱 (민음사)**: 책등(spine) UI — 얇은 막대가 카테고리별로 꽂혀있는 책장. 빈 슬롯이 수집욕을 자극

## 12. 전체 화면 구조 및 유저 흐름

```
[로그인] → [온보딩] → middleware → [홈]
                                     ├─ 글감 카드 클릭 → [에디터] → 저장 후 홈 복귀
                                     ├─ 자유 주제 버튼 → [에디터 (자유)] → 저장 후 홈 복귀
                                     └─ 사이드바 → [내 글 / 설정 / 카테고리 편집]
                                                        └─ 내 글 → [아카이브 책장]
                                                                       └─ 카드 클릭 → [에디터 (수정)]
```

### 유저 흐름 점검 결과 (비판적 검토)

#### 🔴 해결 필요
| 문제 | 해결 방향 |
|---|---|
| 글 쓴 직후 피드백 없음 | 홈 복귀 시 캘린더 오늘 칸 채워지는 즉각 피드백 |
| 자유 주제 글의 아카이브 위치 미정 | writings.topic_id = null → '자유' 별도 섹션 |
| 글감 소진 시 UX 없음 | "모든 글감을 봤어요. 다시 보기" 옵션 추가 필요 |

#### 🟡 보완 필요
| 문제 | 해결 방향 |
|---|---|
| Day 1 경험 빈약 (책장·캘린더 모두 비어있음) | 첫 진입 시 간단한 온보딩 힌트 or 빈 상태 안내 문구 |
| 글 삭제 실수 가능성 | 삭제 확인 모달 필수 |
| 카테고리 편집 진입점 불명확 | 사이드바 + 홈 카테고리 칩 "편집" 탭으로 이중 노출 |

#### 🟢 잘 된 것
- middleware 접근 제어 완벽
- 온보딩 강제 선택 (스킵 없음)
- 디바운스 자동저장 (이탈 방지)
- 새로고침 기능 (이탈 대신 새 글감)
- user_seen_topics로 중복 글감 방지
