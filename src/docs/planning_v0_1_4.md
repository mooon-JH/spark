## 컨텍스트 인계

너는 전문적인 PM이자 개발자야. MVP를 만들기 위해 최대한 단순하고 간소화된 방향으로 개발해야 해. 내 의견을 항상 비판적으로 분석해야 해.

## 1. 기획 진화 흐름

| 단계 | 핵심 변화 | 문제 / 이유 |
|---|---|---|
| 1단계 초기 설계 | 카테고리 10개 / 결핍·후회 중심 톤 | 부정적 감정 편향 / 카테고리 중복 |
| 2단계 타겟 재정의 | 직장인 한정 → 예비 창업자·내면 탐구로 확장 | 카테고리와 글감의 결 전면 재검토 필요 |
| 3단계 글감 설계 원칙 | 긍정·호기심·탐구 톤 확립 | 목적 추가: 글쓰기 습관 + 생각 구조화 훈련 |
| 4단계 카테고리 재편 | 10개 → 16개 (4개 축) | 중복 통합·분리·확장 |
| 5단계 프롬프트 방향 | 단일 → 카테고리별 16개 개별 설계 | 각 카테고리마다 톤·예시·금지어 따로 설계 |
| 6단계 UX 재설계 | 저널/일력 벤치마킹 → 책장 아카이브 도입 | 습관 시각화 + 수집욕 기반 리텐션 설계 |
| 7단계 UX 전면 재설계 | 극효율화·극간소화 → 홈=글감+입력창, 온보딩 제거, 아카이브 재설계 | 사용자 진입 장벽 제거 + 알고리즘 기반 개인화 |

## 2. 제품 개요

### 핵심 가치
- **시작을 쉽게** — 글감이 질문이고 입력창이 답변 공간. 첫 글자를 치면 바로 에디터
- **이탈하지 않게** — 편리한 UX로 흐름 유지. 자동저장
- **쌓이게** — 글이 누적되어 습관화 + 아카이브에서 시각적으로 확인

### 궁극적 목적
글감에서 출발하되, 사용자가 자신의 생각을 구조화하고 글로 표현하게 만드는 것. 쓸수록 나에게 맞는 글감이 나오는 구조.

### 타겟 사용자
| 구분 | 내용 |
|---|---|
| 연령대 | 20대 중반 ~ 30대 중반 |
| 타입 1 | 스타트업 종사자 / 커리어 성장에 관심 있는 직장인 |
| 타입 2 | 창업을 희망하는 사람 (막연히 꿈꾸는 직장인 + 예비 창업자) |
| 타입 3 | 나의 내면을 잘 가꾸어내고 싶은 사람 |
| 페인포인트 | 내 생각을 구조화하는 데 어려움이 있어 글쓰기로 키우고 싶은 사람 |

### 비전 로드맵
| 단계 | 내용 |
|---|---|
| MVP (지금) | 글감 + 입력창으로 쓰기 시작 → 습관 형성 |
| v2 | 사용 데이터 기반 글감 알고리즘 고도화 + 3초 멈춤 → 다음 질문 제안 |
| v3 | 글 구조 피드백 (주장·근거·구조화) → 기획력 훈련 |

## 5. 기준점 (Standards)

### 5-1. 서비스 톤앤매너

**한 줄 정의:** 카페에서 나를 잘 아는 친구가 건네는 말

| 항목 | 기준 |
|---|---|
| 경어 | 해요체 — 합쇼체 X / 반말 X |
| 온도 | 차분하고 따뜻함 — 열정적·명령적 금지 |
| 거리감 | 옆에 앉은 친구 — 코치·선생님·AI 느낌 X |
| 문장 길이 | 짧고 끊어지는 리듬 |
| 느낌표 | 최소 1개 이하 |
| 금지 표현 | "~해보세요!" / "~하면 좋아요!" / AI 챗봇 말투 |
| 버튼 레이블 원칙 | 기능 설명 X → 유저의 심리·행동으로 표현 |
| 넛지 원칙 | 압박 X → 허락·공감으로 |

### 5-2. 글감 톤앤매너

| 항목 | 기준 |
|---|---|
| 형태 | 2인칭 질문형 고정 |
| 어미 | "~한 적 있나요?" / "~이었나요?" / "~했나요?" |
| 길이 | 15~25자 이내 |
| 금지 | 번아웃·탈진·무기력·결핍·후회 전제 / 뻔한 자기계발 / 결론 내버리기 |

### 5-3. 버튼·레이블 기준

| 상황 | 확정 문구 |
|---|---|
| 자유 주제 | "자유롭게 써볼게요" |
| 에러·빈 아카이브 | "아직 쓴 글이 없어요. 첫 글을 써볼까요?" |
| 네트워크 오류 | "연결이 잠깐 끊겼어요. 다시 시도해볼게요." |
| 글감 소진 | "모든 글감을 다 봤어요. 처음부터 다시 볼까요?" |

## 6. ⚠️ DB 스키마 (v4.0 변경사항 포함)

### 변경사항 요약
- `users` 테이블에 `nickname`, `writing_motivation` 컬럼 추가
- `user_categories` 테이블에 `score` 컬럼 추가 (알고리즘 가중치용)
- `user_categories` 의 카테고리 선택 방식 변경 (온보딩 선택 → 알고리즘 자동 학습)

### 전체 스키마

| 테이블 | 주요 컬럼 | RLS |
|---|---|---|
| users | id, email, nickname, writing_motivation, onboarding_completed | 본인만 |
| topics | id, content | 전체 공개 |
| writings | id, user_id, topic_id, topic_content, body | 본인만 |
| user_seen_topics | id, user_id, topic_id | 본인만 |
| topic_categories | topic_id, category | 전체 공개 |
| user_categories | user_id, category, score, click_order, is_removed, selected_at | 본인만 |

### users 테이블 신규 컬럼

| 컬럼 | 타입 | 의미 |
|---|---|---|
| nickname | text | Google 이름 자동 연동. 추후 아카이브에서 변경 가능 |
| writing_motivation | text | 글쓴 계기 선택값. CHECK: complex / unclear / expression / curious |

### user_categories 신규 컬럼

| 컬럼 | 타입 | 의미 |
|---|---|---|
| score | integer | 알고리즘 가중치. 글 저장 +3 / 에디터 진입 +2 / 글감 멈춤 +1 / 글감 넘김 +0 |

### writings 테이블 비고
- `topic_id`가 null이면 자유 주제 글
- `first_sentence` 컬럼 제거 (v3.3)

### SQL 마이그레이션

```sql
-- users 테이블 컬럼 추가
ALTER TABLE users
ADD COLUMN nickname text,
ADD COLUMN writing_motivation text
  CHECK (writing_motivation IN ('complex', 'unclear', 'expression', 'curious'));

-- user_categories score 컬럼 추가
ALTER TABLE user_categories
ADD COLUMN score integer DEFAULT 0;
```

## 7. 글감 알고리즘 (v4.0 신규)

### 개요
카테고리를 온보딩에서 선택하는 방식 → 사용 데이터로 자동 학습하는 방식으로 전환.
넷플릭스처럼 쓸수록 나에게 맞는 글감이 나오는 구조.

### MVP 알고리즘 (가중치 점수 방식)

```
행동별 점수
────────────────────────
글 저장 완료    +3점  (강한 신호)
에디터 진입     +2점  (중간 신호)
글감에서 멈춤   +1점  (약한 신호)
글감 넘김       +0점  (노이즈)
```

```sql
-- 글감 뽑기 쿼리 (가중치 반영)
SELECT t.* FROM topics t
JOIN topic_categories tc ON t.id = tc.topic_id
LEFT JOIN user_categories uc
  ON tc.category = uc.category AND uc.user_id = $1
WHERE t.id NOT IN (
  SELECT topic_id FROM user_seen_topics WHERE user_id = $1
)
ORDER BY (random() * COALESCE(uc.score, 1)) DESC
LIMIT 20;
```

### 글감 노출 비율 (필터 버블 방지)

| 비율 | 대상 |
|---|---|
| 70% | score 높은 카테고리 글감 |
| 20% | score 낮은 카테고리 글감 |
| 10% | 아직 경험 못 한 카테고리 글감 |

### 초기값 (첫 진입 시)
score가 없는 상태에서는 감정 + 커리어 카테고리 글감 우선 노출.
타겟 유저 전반에 걸쳐 반응률이 높은 카테고리. 데이터가 쌓이면 자동 조정.

### v2 이후 로드맵
- 글감 단위 행동 추적 (`user_topic_interactions` 테이블)
- 작성 시간, 에디터 체류 시간 신호 추가
- Collaborative filtering (데이터 충분 후)

## 9. 전체 화면 구조 (v4.0 확정)

### ⚠️ 화면 수 변경: 온보딩 제거 → 4개 화면으로 단순화

```
로그인
  ↓
홈 (= 글감 + 입력창)
  ↓
에디터
  ↓
아카이브
```

### 유저 진입 흐름

```
링크 접속
  ↓
로그인 화면  —  브랜드 + 카피 + Google 버튼
  ↓ Google 로그인
홈  —  글감 + 입력창 바로 노출
  ↓ 첫 글자 입력 즉시
에디터  —  글감 고정 + 본문 입력
  ↓ 첫 글 저장 후
[글쓴 계기 질문]  —  writing_motivation 저장
  ↓
홈  —  "✦ 오늘 칸이 채워졌어요" 피드백 (2~3초 후 사라짐)
```

### 아카이브 진입 흐름

```
홈 상단 아카이브 아이콘 탭
  ↓
아카이브  —  프로필 + 동기부여 + 글 목록
  ↓ 글 탭
에디터  —  기존 글 수정
```

## 11. ⚠️ 온보딩 UX (v4.0 전면 변경)

### 변경 요약

| 항목 | v3.4 (이전) | v4.0 (변경) |
|---|---|---|
| 단계 수 | 1단계 (카테고리 선택) | 0단계 — 온보딩 화면 없음 |
| 카테고리 선택 | 온보딩에서 강제 선택 | 알고리즘 자동 학습으로 대체 |
| 닉네임 | 없음 | Google 자동 연동 (별도 입력 없음) |
| 글쓴 계기 | 없음 | 첫 글 저장 직후 질문 |

### 온보딩 제거 이유
- 극효율화·극간소화 원칙 — 로그인하면 바로 홈
- 카테고리 선택을 알고리즘으로 대체 — 처음부터 정확히 고를 필요 없음
- 닉네임은 Google에서 자동 연동 — 별도 입력 불필요
- 홈 자체가 온보딩 역할 수행

### 글쓴 계기 질문 (첫 글 저장 직후)

```
글이 쓰고 싶어진 건 어떤 순간이었나요?

○ 머릿속이 너무 복잡할 때
○ 내 생각이 뭔지 모를 때
○ 하고 싶은 말이 막힐 때
○ 그냥 써보고 싶었어요
```

### 글쓴 계기 → 경험 변화 연동

| 선택값 | 코드 | 첫 글감 우선 카테고리 | 홈 넛지 | 에디터 placeholder |
|---|---|---|---|---|
| 머릿속이 너무 복잡할 때 | complex | 감정 → 철학 → 인간관계 | "복잡한 게 많은 날엔 그냥 써봐요" | "지금 가장 먼저 떠오르는 장면부터요." |
| 내 생각이 뭔지 모를 때 | unclear | 철학 → 커리어 → 취향 | "쓰다 보면 내가 뭘 원하는지 보여요" | "떠오르는 대로 써봐요. 정답 없어요." |
| 하고 싶은 말이 막힐 때 | expression | 크리에이티브 → 감정 → 컬쳐 | "딱 한 줄이어도 괜찮아요" | "말하듯이 써봐요." |
| 그냥 써보고 싶었어요 | curious | 셔플 (기본값) | "마음에 드는 글감이 나올 때까지 넘겨봐요" | "떠오르는 장면부터 써봐요." |

### 아카이브 첫 진입 시 활용

```
"머릿속이 복잡할 때 시작했죠. 지금은 N편 썼어요."
```

```

| 에디터 전환 | 첫 글자 입력 즉시 전환. 뒤로가기 시 입력 내용 홈에서 유지 |

| 사이드바 | 제거 |
| 캘린더 | 제거 |

**글감 레퍼런스 (Claude·ChatGPT 홈 구조 참고)**

```

상단 — 프로필 (얇게)

### 스트릭 방식 (아카이브에서 담당)

| 항목 | 확정 내용 |
|---|---|
| 방식 | 누적 총편수(절대 감소 없음) + 이번 주 진행도 |
| 주간 리셋 | 매주 월요일 자동 리셋. 미달성 페널티 없음 |
