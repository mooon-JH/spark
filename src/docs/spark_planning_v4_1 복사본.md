## 컨텍스트 인계

너는 전문적인 PM이자 개발자야. MVP를 만들기 위해 최대한 단순하고 간소화된 방향으로 개발해야 해. 내 의견을 항상 비판적으로 분석해야 해.

## 1. 제품 개요

### 핵심 가치
- **시작을 쉽게** — 글감이 질문이고 입력창이 답변 공간. 첫 글자를 치면 바로 에디터
- **이탈하지 않게** — 자동저장 + 이어쓰기로 흐름 유지
- **쌓이게** — 글이 누적되어 습관화. 아카이브에서 시각적으로 확인

### 궁극적 목적
글감에서 출발하되, 사용자가 자신의 생각을 구조화하고 글로 표현하게 만드는 것. 쓸수록 나에게 맞는 글감이 나오는 구조.

### 타겟 사용자

| 구분 | 내용 |
|---|---|
| 연령대 | 20대 중반 ~ 30대 중반 |
| 타입 1 | 스타트업 종사자 / 커리어 성장에 관심 있는 직장인 |
| 타입 2 | 창업을 희망하는 사람 (막연히 꿈꾸는 직장인 + 예비 창업자) |
| 타입 3 | 나의 내면을 잘 가꾸어내고 싶은 사람 |
| 페인포인트 | 내 생각을 구조화하는 데 어려움이 있어 글쓰기로 키우고 싶은 사람 |

### 비전 로드맵

| 단계 | 내용 |
|---|---|
| MVP (지금) | 온보딩 → 글감 추천 → 글쓰기 → 이어쓰기 → 코칭 → 아카이브 |
| v2 | 홈 전면 재설계 + 글감 알고리즘 고도화 + 이어쓰기 자동 트리거 최적화 |
| v3 | 글 구조 피드백 (주장·근거·구조화) → 기획력 훈련 |

## 3. 화면 구조

### 전체 흐름

```
로그인 → 온보딩 1단계 → 온보딩 2단계 → 홈 → 에디터 → 아카이브
```

### 최초 진입 흐름 (온보딩)

```
링크 접속
  ↓
로그인  —  spark 브랜드 + 카피 + Google 버튼
  ↓ Google 로그인 (최초 1회)
온보딩 1단계  —  시작 계기 선택 (4개)
  ↓ 선택 즉시
온보딩 2단계  —  첫 글 쓰기
  ↑ 뒤로가기 → 1단계 복귀
  ↓ 저장 → 홈
  ↓ 뒤로가기(1단계에서) → 홈 (curious 기본값 적용)
  [소개 글 자동 생성 — 온보딩 2단계 진입 시점에 writings 삽입]
  ↓
홈
```

### 반복 사용 흐름

```
홈  —  글감 카드 + 입력창
  ↓ 첫 글자 입력 즉시
에디터  —  글쓰기 + 자동저장 + 이어쓰기 + 코칭
  ↓ 저장 or 뒤로가기
홈 복귀 (자동저장)
```

### 아카이브 흐름

```
홈 상단 아카이브 아이콘
  ↓
아카이브  —  프로필 + 설정 + 글 목록 + 검색
  ↓ 글 탭
에디터 (수정 모드)  —  저장 후 아카이브 복귀
```

### 네비게이션 원칙

- 탭바 없음. 홈 상단 우측 아카이브 아이콘 하나
- 에디터 복귀 분기: 새 글 작성 → 홈 / 기존 글 수정 → 아카이브
- 온보딩 중 뒤로가기: 1단계 → 홈 이탈 / 2단계 → 뒤로가기 없음 (헤더 UI 미표시, 시스템 제스처 무시)

## 4. 온보딩 UX

### 구조
로그인 직후 2단계. 최초 1회만 진행. 이후 재진입 시 홈으로 바로 이동.

**설계 원칙:** 빠르게 첫 글쓰기 경험을 완료시키는 것이 목적. 선택지를 늘리거나 되돌아가는 경로는 모두 차단.

### 전체 흐름

```
로그인
  ↓
1단계 — 시작 계기 선택
  선택 즉시 → 2단계 (router.replace로 history 교체)
  뒤로가기 → 홈 이탈 (curious 기본값)
  ↓
2단계 — 첫 글 쓰기
  [소개 글 자동 생성 — 이 시점에 writings 테이블 삽입]
  뒤로가기 없음 — 저장 후 홈으로만 이동 가능
  ↓
홈 (아카이브에 최소 1편 이상 존재)
```

> ⚠️ 2단계는 `router.replace` 사용. 뒤로가기 버튼 UI 미표시. iOS/Android 시스템 스와이프 제스처 무시.

### 1단계 — 시작 계기 선택

```
spark를 시작한 이유를 알려주시면
나에게 맞는 글감을 먼저 보여드릴게요

○  머릿속이 너무 복잡할 때
○  내 생각이 뭔지 모를 때
○  하고 싶은 말이 막힐 때
○  그냥 써보고 싶었어요     ← 스킵과 동일
```

| 항목 | 내용 |
|---|---|
| 선택 방식 | 탭 즉시 2단계 전환. 별도 확인 버튼 없음 |
| 뒤로가기 | 홈 이탈 (writing_motivation = curious 기본값 적용) |
| 진행 표시 | 없음 — 단계 표시가 오히려 온보딩임을 상기시킴 |

### 2단계 — 첫 글 쓰기

```
[글감 — writing_motivation에 맞는 글감 1개 자동 제시]

[지금 떠오르는 것 아무거나, 한 줄만 써봐요.]
```

| 항목 | 내용 |
|---|---|
| 글감 | writing_motivation에 맞는 카테고리에서 1개 자동 제시. 전환 없음 |
| placeholder | "지금 떠오르는 것 아무거나, 한 줄만 써봐요." — 모든 writing_motivation 동일 |
| 이어쓰기 | 비활성 |
| 코칭 | 비활성 |
| 자동저장 | 활성 (기존 에디터와 동일) |
| 뒤로가기 버튼 | **없음** — 헤더에 뒤로가기 UI 자체 미표시. 시스템 제스처 무시 |
| 탈출 방법 | 없음. 저장 후 홈 진입만 가능 |
| 헤더 | 저장 상태만 표시. 뒤로가기 없음 |

### 선택값 → 경험 연동

| 선택값 | 코드 | 우선 카테고리 | 홈 에디터 placeholder (이후) |
|---|---|---|---|
| 머릿속이 너무 복잡할 때 | complex | 감정 → 철학 → 인간관계 | 지금 가장 먼저 떠오르는 장면부터요. |
| 내 생각이 뭔지 모를 때 | unclear | 철학 → 커리어 → 취향 | 떠오르는 대로 써봐요. 정답 없어요. |
| 하고 싶은 말이 막힐 때 | expression | 크리에이티브 → 감정 → 컬쳐 | 말하듯이 써봐요. |
| 그냥 써보고 싶었어요 | curious | 셔플 (기본값) | 떠오르는 장면부터 써봐요. |

> 온보딩 2단계 placeholder는 위 표와 무관하게 단일 문구 고정.

### Spark 소개 글 (자동 생성)

2단계 진입 시점에 서버에서 자동 삽입. 글 작성 여부와 무관하게 생성.

| 항목 | 내용 |
|---|---|
| topic_id | null |
| topic_content | null |
| body | "오늘 spark를 시작했어요. 앞으로 여기서 쌓여갈 생각들이 기대돼요. ✦" |
| is_system | true |
| 알고리즘 영향 | 없음 (카테고리 없음) |
| 삭제 | 가능. 모달 문구: "spark의 첫 날이 기록된 글이에요. 정말 삭제할까요?" |

### middleware 처리

| 조건 | 처리 |
|---|---|
| 비로그인 | /login 리다이렉트 |
| 로그인 + writing_motivation null | /onboarding 리다이렉트 |
| 로그인 + writing_motivation 있음 | / 홈 |
| 기존 유저 (onboarding_completed = true, writing_motivation null) | /onboarding 1회 진행 |

```
─────────────────────────────
OOO님, 안녕하세요   [아카이브 아이콘]
─────────────────────────────

[←]  잘하고 싶은 마음이 오히려 발목을 잡은 적 있나요?  [→]

[떠오르는 장면부터 써봐요...]
```

### 확정 항목

| 항목 | 내용 |
|---|---|
| 상단 | 닉네임 인사 (Google 자동 연동) + 아카이브 아이콘 |

| 글감 일일 제한 | 하루 20개. 초과 시 "오늘의 글감을 모두 봤어요. 내일 새로운 글감이 기다리고 있어요." + 자유롭게 써볼게요 버튼 |
| 글감 피드백 | 글감 하단 👍 👎 아이콘 (작게, 선택). 좋아요 +2 / 싫어요 -2. 싫어요 시 user_seen_topics 추가 (30일 후 만료) |
| 입력창 | 글감 바로 아래. placeholder: writing_motivation에 따라 변경 |
| 에디터 전환 | 첫 글자 입력 즉시 전환 |
| 자유 주제 | 입력창 아래 텍스트 링크 "자유롭게 써볼게요" |
| 네비게이션 | 탭바 없음. 상단 아카이브 아이콘 하나 |
| 스트릭 | 홈에서 표시 없음. 아카이브에서 담당 |

## 6. 에디터 UX

### 기본 스펙

| 항목 | 내용 |
|---|---|
| 기기 | 모바일 우선 / 최대 너비 390px |
| 글감 위치 | 본문 위 — 스크롤 내려가면 자연스럽게 사라짐 |
| 글감 수정 | 가능 (원본 topic_id 고정 저장) |
| placeholder | writing_motivation에 따라 변경 |
| 글자 수 | 제한 없음 |
| 서식 | 볼드 / 이탤릭 툴바 최소한만. 키보드 위 고정 |
| 자동 저장 | 타이핑 멈추고 1.5초 후 디바운스 저장 |
| 저장 상태 | 헤더 우측 "저장 중..." / "✓ 저장됨" / "저장 실패" |
| 이탈 | 뒤로가기 → 즉시 저장 후 홈 복귀 |

### 이어쓰기

**핵심 경험:** 사용자가 글을 쓰다 멈추는 찰나에, AI가 다음 문장을 회색 텍스트로 조용히 제안. 쓰는 흐름을 깨지 않는 것이 최우선.

```
타이핑 멈춤 (3~4초)
  ↓
백그라운드 API 호출 (사용자 모름)
  ↓
커서 뒤에 회색 흐릿한 텍스트로 1문장 제안
  ↓
✓ 수락 → 한 글자씩 타이핑 효과로 본문 합류
✕ 거절 → 제안 텍스트 사라짐
```

| 항목 | 내용 |
|---|---|
| 멈춤 감지 시간 | 3~4초 시작. 데이터 보고 조정 |
| 컨텍스트 범위 | 현재 문단 전체 + 직전 문단 첫 문장 |
| 제안 수 | 1문장 |
| 표시 | 커서 뒤 회색 흐릿한 텍스트 (opacity 낮게) |
| 수락 | 하단 고정 ✓ 버튼 (문구 없음) |
| 거절 | 하단 고정 ✕ 버튼 (문구 없음) |
| 수락 애니메이션 | 한 글자씩 타이핑 효과 |
| 수락 후 쿨다운 | 10초. 이후 멈춤 감지 재활성 |
| 첫 등장 툴팁 | "AI가 다음 문장을 제안했어요. ✓ 수락 ✕ 거절. 원하지 않으면 아카이브에서 끌 수 있어요." → 탭하거나 3초 후 자동 소멸 |
| 이후 | 버튼만 표시 (문구 없음) |
| 최소 글 길이 | 제한 없음 — 짧은 글에서도 작동. 흐름 경험이 목적 |
| API 실패 시 | 조용히 무시. 다음 멈춤 때 재시도 |
| 비용 절감 | 현재 문단 + 직전 문단 첫 문장만 전송. max_tokens 100 |

### 코칭

| 항목 | 내용 |
|---|---|
| 트리거 | 에디터 하단 "✦ 흐름 보기" 버튼. 50자 이상일 때 활성화 |
| 컨텍스트 | 글 전체 본문 |
| 결과 표시 | 논모달 바텀 시트 (드래그 가능. 배경 dimming 없음) |
| 기본 높이 | 화면 40%. 드래그업 시 80% |
| 비용 | 사용자가 원할 때만 호출 |
| 피드백 톤 | 공감 우선 → 평가 금지. 짧고 감정적인 글에도 "잘 됐다/못 됐다" 판단 없이 공감으로 시작 |
| API 실패 시 | "잠깐 문제가 생겼어요. 다시 시도해볼게요." + 재시도 버튼 |

## 7. 아카이브 UX

### 역할 정의
홈이 글쓰기에만 집중하는 공간이라면, 아카이브는 **홈에서 처리하지 않는 모든 것을 담는 공간**이다. 프로필, 설정, 통계, 내 글 목록, 검색이 모두 여기에 있다. 마이페이지처럼 보이는 것이 의도된 구조다.

### 레이아웃

```
─────────────────────────────
상단 — 프로필 섹션
  [Google 사진]  OOO님       로그아웃
  닉네임 수정 (연필 아이콘)
  총 N편 썼어요  ·  이번 주 N편
  이어쓰기 제안  [토글 ON/OFF]
─────────────────────────────
하단 — 글 목록
  [검색창]

  제목 또는 첫 줄 + 미리보기 2줄     5월 6일
  제목 또는 첫 줄 + 미리보기 2줄     5월 4일
  제목 또는 첫 줄 + 미리보기 2줄     5월 1일
─────────────────────────────
```

### 프로필 섹션

| 항목 | 내용 |
|---|---|
| 사진 | Google 프로필 이미지 자동 연동 |
| 닉네임 | Google 이름 자동 연동. 연필 아이콘 탭 → 인라인 수정 → 확인/취소 |
| 통계 | "총 N편 썼어요 · 이번 주 N편". 주간 리셋 월요일. 미달성 페널티 없음 |
| 마지막 작성일 | "마지막으로 쓴 날: N일 전". 재진입 사용자 맥락 제공. 오늘 쓴 경우 표시 안 함 |
| 로그아웃 | 프로필 우측. 확인 모달 후 로그인 화면으로 |

### 글 목록

| 항목 | 내용 |
|---|---|
| 표시 방식 | 제목(없으면 첫 줄) + 미리보기 2줄 + 날짜 |
| 카테고리 표시 | 없음 — 카테고리는 백엔드 알고리즘에만 사용. 사용자에게 노출 안 함 |
| 탭 필터 | 없음 |
| 검색 | 글 10편 이상일 때 검색창 노출. writings.body + topic_content 전문 검색 |
| 정렬 | 최신순 고정 |
| 글 탭 | 에디터로 진입 (수정 모드). 저장 후 아카이브로 복귀 |
| 글 삭제 | 스와이프 or ··· 메뉴 → 삭제 확인 모달 |
| 소개 글 삭제 모달 | "spark의 첫 날이 기록된 글이에요. 정말 삭제할까요?" |
| 빈 상태 | "아직 쓴 글이 없어요. 첫 글을 써볼까요?" + 홈 이동 버튼 |

### 에디터 복귀 분기

| 진입 경로 | 저장 후 복귀 |
|---|---|
| 홈 → 에디터 (새 글) | 홈으로 복귀 |
| 아카이브 → 에디터 (수정) | 아카이브로 복귀 |

### 제거된 항목 (v4.2 변경)

| 항목 | 제거 이유 |
|---|---|
| 카테고리 진행도 바 | 카테고리 미노출 방침과 불일치 |
| 카테고리 탭 필터 | 동일 |
| 카테고리 조정하기 링크 | 카테고리 직접 선택 온보딩 제거로 불필요 |
| 쿠폰 효과 (2칸 미리 채움) | 진행도 바 제거로 함께 제거 |

## 8. 글감 알고리즘

### 개요
카테고리 16개는 고정. 사용자 행동 데이터로 자동 학습하여 개인화. 쓸수록 나에게 맞는 글감이 나오는 구조.

### 행동별 점수

```
글 저장 완료    +3점  (강한 신호)
에디터 진입     +2점  (중간 신호)
글감 좋아요     +2점  (직접 피드백)
글감 싫어요     -2점  (직접 피드백)
글감 넘김       +0점  (노이즈)
```

> ⚠️ MVP에서는 글 저장 +3 / 에디터 진입 +2 / 좋아요·싫어요 ±2만 구현. 멈춤 감지 신호는 v2.

### 글감 뽑기 쿼리

```sql
SELECT t.* FROM topics t
JOIN topic_categories tc ON t.id = tc.topic_id
LEFT JOIN user_categories uc
  ON tc.category = uc.category AND uc.user_id = $1
WHERE t.id NOT IN (
  SELECT topic_id FROM user_seen_topics WHERE user_id = $1
)
ORDER BY (random() * COALESCE(uc.score, 1)) DESC
LIMIT 20;
```

### 글감 노출 비율

| 비율 | 대상 |
|---|---|
| 70% | score 높은 카테고리 글감 |
| 20% | score 낮은 카테고리 글감 |
| 10% | 아직 경험 못 한 카테고리 글감 |

### 초기값 (첫 진입 시)
`writing_motivation` 선택값에 따라 우선 카테고리 결정. score 없는 상태의 콜드 스타트 해소.

## 9. DB 스키마

### 전체 테이블

| 테이블 | 주요 컬럼 | RLS |
|---|---|---|
| users | id, email, nickname, writing_motivation, onboarding_completed | 본인만 |
| topics | id, content | 전체 공개 |
| writings | id, user_id, topic_id, topic_content, body, is_system | 본인만 |
| user_seen_topics | id, user_id, topic_id, expires_at, seen_at | 본인만 |
| topic_categories | topic_id, category | 전체 공개 |
| user_categories | user_id, category, score, click_order, is_removed, selected_at | 본인만 |

### v4.2 신규 / 변경 컬럼

| 테이블 | 컬럼 | 타입 | 의미 |
|---|---|---|---|
| users | nickname | text | Google 이름 자동 연동. 아카이브에서 변경 가능 |
| users | writing_motivation | text | 온보딩 선택값. complex / unclear / expression / curious |
| user_categories | score | integer | 알고리즘 가중치. 기본값 0 |
| writings | is_system | boolean | 시스템 자동 생성 글 여부. 기본값 false. Spark 소개 글에만 true |
| user_seen_topics | expires_at | timestamptz | 싫어요로 숨긴 글감 만료일. NULL이면 영구 제외 (글 작성한 글감). 30일 후 만료 (싫어요 글감) |
| user_seen_topics | seen_at | timestamptz | 글감을 넘긴 시각. 일일 제한(20개) 계산에 사용 |

### writings 테이블 비고
- `topic_id`가 null이면 자유 주제 글 또는 시스템 생성 글
- `topic_content`는 글감 스냅샷 — 원본 글감이 수정·삭제되어도 작성 당시 글감 보존
- `is_system = true`인 글은 삭제 시 별도 모달 문구 적용

### user_seen_topics 비고
- `expires_at = null` → 영구 제외 (글 작성한 글감, 글감 소진 후 리셋 대상)
- `expires_at = 30일 후` → 싫어요 누른 글감. 만료 후 자동 재노출
- 일일 제한 쿼리: `seen_at >= today` 기준으로 하루 20개 초과 여부 판단

### SQL 마이그레이션

```sql
-- users 테이블 컬럼 추가
ALTER TABLE users
ADD COLUMN nickname text,
ADD COLUMN writing_motivation text
  CHECK (writing_motivation IN ('complex', 'unclear', 'expression', 'curious'));

-- user_categories score 컬럼 추가
ALTER TABLE user_categories
ADD COLUMN score integer DEFAULT 0;

-- writings is_system 컬럼 추가
ALTER TABLE writings
ADD COLUMN is_system boolean DEFAULT false;

-- user_seen_topics 컬럼 추가
ALTER TABLE user_seen_topics
ADD COLUMN expires_at timestamptz DEFAULT NULL,
ADD COLUMN seen_at timestamptz DEFAULT NOW();
```

### 글감 일일 제한 쿼리

```sql
-- 오늘 넘긴 글감 수 확인
SELECT COUNT(*) FROM user_seen_topics
WHERE user_id = $1
AND seen_at >= CURRENT_DATE
AND seen_at < CURRENT_DATE + INTERVAL '1 day';
-- 20개 이상이면 더 이상 넘기기 불가
```
